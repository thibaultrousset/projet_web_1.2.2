Introduction

Un framework CSS est un ensemble de composants structurés de sorte à faciliter et standardiser le web-design en utilisant le language CSS. La plupart des ces frameworks contiennent au moins une grille, et les plus fonctionnels possèdent d'autres fonctionnalités, liées par exemple à Javascript, bien qu'ils soient surtout orientés design. Ceci les différencie de frameworks dédiés à Javascript.

Bootstrap

Qu'est-ce que Bootstrap?

Bootstrap est un framework CSS destiné à faciliter la création d'applications web, composé d'un ensemble de classes CSS et de librairies javascript.

Ce framework, composé de feuilles de style CSS, contient des définitions de base pour tous les éléments HTML, ce qui permet d'uniformiser l'apparence des textes, tableaux et autres éléments de formulaires.
Il standardise également de nombreux éléments graphiques, tel que les boutons, libellés, icônes et bien d'autres.

L'histoire de Bootstrap

Initialement développé en 2010 par Mark Otto et Jacob Thornton, deux développeurs travaillant pour Twitter, comme solution à usage interne pour résoudre les incohérences de développements de l'équipe d'ingénieurs de Twitter . Il est devenu le frameworks CSS de référence, ainsi qu'un des projets les plus populaires de la plateforme de gestion de développement Git-hub.

Depuis la version 2, il est possible de créer des designs "responsives", permettant au contenu de s'adapter à la taille de l'écran.

Les indispensables

Pour inclure Bootstrap dans un site web, il y a deux méthodes:

  1. télécharger les fichier CSS depuis "getbootstrap.com"
  2. inclure Bootstrap depuis un CDN (Content Delivery Network)

Bootstrap utilise des éléments HTML et des propriétés CSS qui nécessitent le doctype HTML5.

Bootstrap 3 est développé principalement pour les mobiles; il est donc important d'ajouter la balise <meta> suivante à l'intérieur de l'élément <head> afin de s'assurer que les fonctions tactiles fonctionnent correctement.

<meta name="viewport" content="width=device-width, initial-scale=1">

La partie "width=device-width" rends la largeur de la page égale à celle de l'écran sur lequel elle est visionnée.
La partie "initial-scale=1" définit le zoom initial lors du chargement de la page par le navigateur.

Conteneurs

Les conteneurs sont la méthode la plus basique de positionnement de Bootstrap et sont requis pour utiliser son système de grille. Deux options responsives sont disponibles :
Un conteneur à largeur fixe (container), dont la largeur maximum (max-width) change à chaque breakpoint.

<div class="container"> <-- Content here --> </div>

Un conteneur à largeur fluide (container-fluid) qui occupera tout l'espace horizontal disponible en permanence.

<div class="container-fluid"> <-- Content here --> </div>

Responsive Breakpoints

Comme Bootstrap est développé en priorité pour les mobiles, il fait usage de media-queries pour créer des breakpoints sensés pour l'affichage. Ces breakpoints sont surtout basés sur des largeurs minimums de fenêtres d'affichage et permet de changer l'échelle des éléments quand la fenêtre change.

Bootstrap utilise principalement les media query 'range' (ou breakpoints) dans ses fichiers sources Sass pour sa mise en page, son système de grille, et ses composants.

// Très petits appareils (téléphone en affichage portrait, moins de 576px)
// Pas de media query puisqu'il est par défaut en Bootstrap

// Petits appareils (téléphone en affichage paysage, 576px et plus)
@media (min-width: 576px) { ... }

// Appareils de taille moyenne (tablette, 768px et plus)
@media (min-width: 768px) { ... }

// Appareils de grande taille (ordinateur portable, 992px et plus)
@media (min-width: 992px) { ... }

// Appareils de très grande taille (ordinateur de bureau, 1200px et plus)
@media (min-width: 1200px) { ... }

L'index Z

Un grand nombre de composants Bootstrap utilisent z-index, la propriété CSS qui aide à contrôler la mise en page en fournissant un troisième axe pour déplacer le contenu. Bootstrap utilise par défaut un index z qui a été conçu pour disposer correctement la navigation, les infobulles, pop-ups et autres.

Il est déconseillé de modifier ces valeurs; Si une est changée, il faudra probablement modifier toutes les autres.

//$zindex-dropdown: 1000 !default;//
//$zindex-sticky: 1020 !default;//
//$zindex-fixed: 1030 !default;//
//$zindex-modal-backdrop: 1040 !default;//
//$zindex-modal: 1050 !default;//
//$zindex-popover: 1060 !default;//
//$zindex-tooltip: 1070 !default;//

Les éléments de fond ont tendance à résider à des index z inférieurs, tandis que la navigation et les infobulles utilisent des index z supérieurs pour s'assurer qu'ils se placent au dessus d'autres contenus.

Les grilles

Bootstrap divise nativement l'écran en 12 colonnes, et permet de redéfinir leur taille en affectant la largeur de certaines de ces colonnes à d'autre.
Il est possible d'en assigner davantage, mais dans un tel cas le trop-plein bascule sur la ligne suivante.

Par exemple, un conteneur auquel on affecterait les classes col-sm-8 prendrait 8 des 12 colonnes d’une ligne sur un écran au format tablette. Si on rajoutait une colonne col-sm-5 dans cette même ligne, on arriverait à une valeur de 13 colonnes; comme elle ne rentrerait pas, elle passerait donc automatiquement à la ligne suivante.

Ces colonnes se placent dans des lignes, des éléments possédant la classe row ainsi que la classe container ou container-fluid. Elles ne doivent contenir que des colonnes, qui elles abriteront le contenu.
Bien que ce découpage en colonne soit une division en pourcentage de la largeur de la fenêtre, les lignes fonctionnent différemment, ne prenant que la hauteur du plus gros élément qu'elles contiennent.
Lors de la création d'une colonne, on indique le nombre de colonnes que l'on voudrait qu'elle occupe à l'aide de la classe col, suivi de la classe correspondant à la taille d'écran que l'on souhaite affecter puis d'un nombre inférieur ou égal à 12.

Par exemple, en utilisant les classes col-md-4 on pourrait créer trois colonnes prenant chacune un tier (4/12) d'un écran de taille moyenne.

Ces classes affectent les écrans de tailles supérieures ou égales à leur "breakpoint", et prennent le pas sur celles affectant les écrans de tailles inférieures.

Une colonne col-sm-12 col-xl-6 prendra toute une ligne sur tablette, mais seulement la moitié sur grand écran.

Afin d'effectuer les modifications présentées plus tôt, quatre classes sont mises à disposition de l'utilisateur:

  • xs pour les écrans de moins de 768 px de large (format mobile)
  • sm pour les écrans d'au moins 768 px de large (format tablette)
  • md pour les écrans d'au moins 992 px de large (format ordinateur portable)
  • lg pour les écrans d'au moins 1200 px de large (format ordinateur de bureau)

    • Ce système de grille étant "responsive", les colonnes se réarrangent automatiquement en fonction de la taille de l'écran.

    Bootstrap met à disposition de l'utilisateur de nombreuses classes affectant la grille de manière plus ou moins apparente; parmis celles-ci, on retrouve la classe offset permettant d'ignorer des colonnes, ainsi que les classes push et pull qui ont pour fonction de décaler une colonne vers la droite ou la gauche, respectivement.

    La documentation Bootstrap étant disponible gratuitement, il est fortement conseillé de se tourner vers elle en cas de question sur les possibilités offertes par ce framework.
